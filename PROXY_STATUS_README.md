# CC API Proxy 状态检查工具

`proxy_status.py` 是一个用于检查 CC API Proxy 运行状态的工具脚本。

## 功能特性

1. **端口检测** - 检查代理是否在端口 5678 上运行
2. **翻译测试** - 发送测试请求验证代理能否正常翻译
3. **统计信息** - 显示当前 provider、模型、连接状态等
4. **彩色输出** - 使用 ANSI 颜色代码显示状态表格
5. **退出码** - 返回适当的退出码供脚本集成使用

## 使用方法

```bash
# 基本用法
python proxy_status.py

# 输出示例（代理运行中）:
正在检查 CC API Proxy (http://127.0.0.1:5678)...
==============================================================
                      CC API Proxy 状态检查                       
==============================================================
检查时间: 2026-04-01 12:52:38
--------------------------------------------------------------
代理状态: running
Provider: deepseek
模型: deepseek-chat
翻译测试: 通过 (1555.25ms)
连接状态: 稳定运行 (3/3) [8.28ms]
端口 5678: 开放
服务器: BaseHTTP/0.6 Python/3.13.12
==============================================================

✓ 代理运行正常

# 退出码:
# 0 - 代理运行正常
# 1 - 代理未运行或存在问题
```

## 检测项目

脚本会检查以下项目：

1. **端口状态** - 检查 5678 端口是否开放
2. **代理信息** - 获取当前 provider 信息
3. **模型列表** - 获取支持的模型信息
4. **翻译功能** - 发送测试请求验证翻译功能
5. **连接稳定性** - 测试连接稳定性并计算平均响应时间

## 集成使用

可以在脚本或 CI/CD 流程中使用：

```bash
# 检查代理状态
if python proxy_status.py; then
    echo "代理运行正常"
else
    echo "代理未运行，正在启动..."
    python api_proxy.py --provider deepseek --port 5678 &
fi

# 获取状态信息（静默模式）
python proxy_status.py 2>/dev/null | grep "代理状态"
```

## 技术细节

- 使用 ANSI 颜色代码，支持 Windows 10+ 和 Unix-like 系统
- 自动检测终端颜色支持
- 超时处理：端口检测 1秒，HTTP请求 3-10秒
- 支持代理的各种状态：运行中、未运行、间歇性连接

## 文件位置

- 主脚本: `D:/CC/cloud-code/proxy_status.py`
